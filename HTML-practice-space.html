CSS Positioning

Static: Boxes are what they are.

Relative: boxes can be nudged relative to their static position. E.g. push it up, down, left, right by a certain number of pixels from where it otherwise would have been.

Absolute: position relative to nearest position ancestor/parent (e.g. when it is nested). So if parent box has a lower bound at 400px,

Fixed: positioned to exact coordinates in the browser window. This is the most sensible way. Positions are not inherited from ancestors.

MEDIA QUERIES

Allows the medium to ask the code to tell it how to format the layout

Phones and tablets viewport = screen, PCs and Macs viewport = browser.

Media queries must be declared last or they get overwritten

The @media query says "if its a medium and a screen, and the max width is x amount of pixels or less, the body background is going to change by y (e.g. the color changes and the wrapper size changes).

Declared in CSS code and looks someting like this:

@media screen and (max-width:768px) {

body {

background-color: #333;

}

.wrapper {

width: 600px;

}

display: none;

}


Use the "toggle device toolbar" in browser f12 and you can change the test screen to "responsive" and adjust the dimensions to test your code.
